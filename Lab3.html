Views & Remote User-Defined Functions

Welcome! In this lab, you will learn how to extend Cloud Spanner query capabilities using **Remote User-Defined Functions (UDFs)** and encapsulate complex calculations using database **Views**.

Overview of Remote UDFs

In traditional databases, UDFs execute inside the database engine. In Cloud Spanner, a **Remote UDF** allows you to trigger external application logic (such as a Google Cloud Function) directly from within a SQL query. Spanner automatically groups rows and sends them in batches to your external endpoint, ensuring high throughput and minimal latency.

Instance Sizing Requirement (Low Cost Instances)
Google Cloud Spanner Remote UDF execution is **not supported on Low Cost Instances** (instances sized below 1 node or 1000 processing units).

If your multi-region-spanner instance is currently configured with 100 processing units (as configured in Lab 2), you must scale it up to 1000 processing units (1 node) to complete this lab: 
gcloud spanner instances update multi-region-spanner \
  --processing-units=1000
Note: You will scale the instance back down to 100 processing units at the end of this lab to minimize project resource consumption and GCP credits usage.
External Computation
Offloads complex processing (like custom cryptography, formatting, or third-party service calls) from the database instances to serverless Cloud Functions.
Virtual Tables (Views)
Views present a virtual table representation of a query. You can wrap complex remote function calls inside a view to present clean, formatted data fields to application clients.

Batch RPC Protocol

Spanner interacts with your Cloud Function using a standard JSON payload format containing a list of arguments (referred to as calls). Your HTTP function must process these batch rows sequentially and respond with a matching list of computed values (referred to as replies).

Step 1: Deploy the Node.js Cloud Function

In this step, you will inspect the Node.js source code for an HTTP Cloud Function that formats numbers into USD currency and deploy it using the gcloud SDK.

1. Review the Node.js Function Code

Below is the Cloud Function implementation using the Google Cloud Functions Framework. Note how it processes the batch request payload structures:

Node.js (index.js)
Copied! 
const functions = require('@google-cloud/functions-framework');

functions.http('formatCurrency', (req, res) => {
  const requestBody = req.body;
  const calls = requestBody.calls || [];
  
  // Process batch rows: each argument set is wrapped in an array
  const replies = calls.map(args => {
    const amount = parseFloat(args[0]);
    if (isNaN(amount)) return null;
    return new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' }).format(amount);
  });
  
  res.status(200).json({ replies });
});

2. Deploy the HTTP Endpoint

Run the following command in your terminal shell to deploy this code to Google Cloud Functions:

Terminal (Bash / PowerShell)
Copied! 
gcloud functions deploy formatCurrency \
  --runtime=nodejs20 \
  --trigger-http \
  --allow-unauthenticated \
  --entry-point=formatCurrency \
  --region=us-central1
Retrieve Endpoint URL
Upon successful deployment, the console output will display the active https endpoint. Copy this URL (e.g. https://us-central1-your-project.cloudfunctions.net/formatCurrency). You will need it in the next step.
  • Deploy the Cloud Function HTTP Endpoint Run the gcloud deployment script and record the returned endpoint URL.

Step 2: Register the Remote UDF in Spanner

Once your Cloud Function is live, you must register it as a custom database function in your Spanner database using DDL syntax.

DDL for Remote Function Registration

Register the UDF inside your database. Remember to replace [YOUR_ENDPOINT_URL] with your actual deployed Cloud Function URL:

DDL SQL
Copied! 
CREATE SCHEMA udf;

CREATE OR REPLACE FUNCTION udf.formatCurrency (amount NUMERIC) RETURNS STRING
  NOT DETERMINISTIC
  LANGUAGE REMOTE
  OPTIONS (
    endpoint = 'https://us-central1-[PROJECT_ID].cloudfunctions.net/formatCurrency'
  );

Execute the DDL Update

Apply the DDL statement to your test-database in the multi-region-spanner instance:

Terminal (Bash / PowerShell)
Copied! 
gcloud spanner databases ddl update test-database \
  --instance=multi-region-spanner \
  --ddl="CREATE SCHEMA udf" \
  --ddl="CREATE OR REPLACE FUNCTION udf.formatCurrency (amount NUMERIC) \
    RETURNS STRING \
    NOT DETERMINISTIC \
    LANGUAGE REMOTE \
    OPTIONS ( \
      endpoint = 'https://us-central1-[PROJECT_ID].cloudfunctions.net/formatCurrency' \
    )"
  • Register the remote function in Spanner Apply the CREATE FUNCTION DDL to test-database pointing to the cloud function endpoint.

Step 3: Querying Data Using the Remote UDF

With the function registered in Spanner's catalog, you can call it directly inside standard SQL SELECT queries like any built-in scalar function.

1. Add Balance Column to TradingAccounts Table

Ensure your TradingAccounts table has a NUMERIC column to represent balances. Apply this schema modification:

Terminal (Bash / PowerShell)
Copied! 
gcloud spanner databases ddl update test-database \
  --instance=multi-region-spanner \
  --ddl="ALTER TABLE TradingAccounts ADD COLUMN Balance NUMERIC"

2. Populate Balances

Update sample rows with positive amounts to format:

Terminal (Bash / PowerShell)
Copied! 
gcloud spanner databases execute-sql test-database \
  --instance=multi-region-spanner \
  --sql="UPDATE TradingAccounts SET Balance = 12500.75 WHERE AccountId = 10"

gcloud spanner databases execute-sql test-database \
  --instance=multi-region-spanner \
  --sql="UPDATE TradingAccounts SET Balance = 945000.50 WHERE AccountId = 11"

3. Run Query with the Remote UDF

Query the table and format the balance values using the UDF:

Terminal (Bash / PowerShell)
Copied! 
gcloud spanner databases execute-sql test-database \
  --instance=multi-region-spanner \
  --sql="SELECT AccountId, AccountName, udf.formatCurrency(Balance) AS FormattedBalance FROM TradingAccounts"
  • Execute SQL Query with UDF Query balances and verify that values are formatted as currency strings ($12,500.75) returned by your Cloud Function.

Step 4: Create a Database View

In this step, you will encapsulate the UDF formatting query into a reusable **Database View** to simplify query access for client applications.

1. Define the View (DDL)

Run the DDL update command to create a view named AccountBalancesView:

Terminal (Bash / PowerShell)
Copied! 
gcloud spanner databases ddl update test-database \
  --instance=multi-region-spanner \
  --ddl="CREATE OR REPLACE VIEW AccountBalancesView \
    SQL SECURITY INVOKER \
    AS SELECT \
      TradingAccounts.AccountId, \
      TradingAccounts.AccountName, \
      udf.formatCurrency(TradingAccounts.Balance) AS FormattedBalance \
    FROM TradingAccounts"

2. Query the View

Select records directly from the database view. Spanner evaluates the underlying UDF on-the-fly:

Terminal (Bash / PowerShell)
Copied! 
gcloud spanner databases execute-sql test-database \
  --instance=multi-region-spanner \
  --sql="SELECT * FROM AccountBalancesView"
