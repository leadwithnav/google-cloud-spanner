Spring Boot & Spring Data Spanner Integration

Welcome! In this hands-on lab, you will learn how to integrate Java applications with Cloud Spanner using Spring Boot and Spring Data Spanner, maps tables to Java classes, and configure session pools.

The Integration Stack

Developing enterprise microservices with Cloud Spanner is simplified by using the Spring Boot framework. Let's look at the component hierarchy of this stack:

Integration Stack Hierarchy 
      Spring Boot Application (Runtime & Web Rest Controller)
                 │
      Spring Data Spanner (Repository Mapping abstractions)
                 │
      Google Cloud Spanner Java Client (Native Under-the-Hood SDK)
                 │
      Session Pool (Local HTTP/2 gRPC Channel Manager)
                 │
      Cloud Spanner (Target Database Instance)

Spring Data Spanner acts as a mapping layer on top of the native **Google Cloud Spanner Java Client**. This client manages the **Session Pool** to coordinate transactions and queries. The configuration parameters we set in Spring Boot properties are mapped directly down to the session pool configuration class of the native client library.

  • Analyze integration components Review the structural delegation from Rest Controllers down to gRPC session pools.

Understanding Component Roles

Let's examine the role of each layer in this integration:

Spring Data Spanner
Maps database tables directly to Java objects using annotations (like @Table, @PrimaryKey). Provides CRUD repositories so developers can execute basic database operations without writing custom SQL.
Google Cloud Java Client
The underlying low-level Google Cloud SDK. It establishes HTTP/2 connections, handles authentication, routes gRPC requests to Spanner splits, and maintains the database session pool.
  • Differentiate framework vs native layers Understand how Spring Data delegates low-level connection management to the Google Cloud Java client.

Step 1: Project Maven Setup

To integrate Spring with Spanner, we include the Spring Cloud GCP starter dependency in our project configuration.

Add Starter Dependencies

Add the following Bill-of-Materials (BOM) management and starter dependency definitions to your pom.xml file:

pom.xml
Copied! 
<!-- Dependency Management for Spring Cloud GCP -->
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.google.cloud</groupId>
            <artifactId>spring-cloud-gcp-dependencies</artifactId>
            <version>4.8.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<dependencies>
    <!-- Spanner Spring Boot Starter -->
    <dependency>
        <groupId>com.google.cloud</groupId>
        <artifactId>spring-cloud-gcp-starter-data-spanner</artifactId>
    </dependency>
    <!-- Spring Boot Web Starter -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>

Step 2: Configuring Database & Session Pools

Configure database connection details and session pool configurations in your Spring Boot application properties file.

Configure application.properties

Add these configuration keys to your src/main/resources/application.properties file. These settings direct Spring to connect to your local Spanner emulator and configure connection pool limits:

application.properties
Copied! 
# Google Cloud Spanner Target Setup
spring.cloud.gcp.spanner.project-id=test-project
spring.cloud.gcp.spanner.instance-id=test-instance
spring.cloud.gcp.spanner.database=test-database

# Emulator override (routes queries to the emulator on port 9010)
spring.cloud.gcp.spanner.emulator-host=localhost:9010

# Session Pool Configurations (mapped down to the Java client library)
spring.cloud.gcp.spanner.session-pool.min-sessions=50
spring.cloud.gcp.spanner.session-pool.max-sessions=200
spring.cloud.gcp.spanner.session-pool.write-fraction=0.25
spring.cloud.gcp.spanner.session-pool.keep-alive-interval-minutes=20

Step 3: Creating Entities & Repositories

Spring Data Spanner maps tables to Java classes using annotations, and handles database operations using repository interfaces.

1. Define the Entity Class

Create a class named Customer.java and map it to your database schema using Spring Data annotations:

Customer.java
Copied! 
package com.example.spanner.model;

import com.google.cloud.spring.data.spanner.core.mapping.Table;
import com.google.cloud.spring.data.spanner.core.mapping.PrimaryKey;
import com.google.cloud.spring.data.spanner.core.mapping.Column;

@Table(name = "Customers")
public class Customer {

    @PrimaryKey
    @Column(name = "CustomerId")
    private Long customerId;

    @Column(name = "Name")
    private String name;

    @Column(name = "Email")
    private String email;

    // Constructors
    public Customer() {}
    
    public Customer(Long customerId, String name, String email) {
        this.customerId = customerId;
        this.name = name;
        this.email = email;
    }

    // Getters and Setters
    public Long getCustomerId() { return customerId; }
    public void setCustomerId(Long customerId) { this.customerId = customerId; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
}

2. Declare the Repository Interface

Create an interface named CustomerRepository.java extending SpannerRepository. This interface automatically provides standard CRUD methods (like save() and findById()) and generates custom query methods based on their signatures:

CustomerRepository.java
Copied! 
package com.example.spanner.repository;

import com.example.spanner.model.Customer;
import com.google.cloud.spring.data.spanner.repository.SpannerRepository;
import org.springframework.stereotype.Repository;
import java.util.List;

@Repository
public interface CustomerRepository extends SpannerRepository<Customer, Long> {
    
    // Spring Data automatically generates the query logic based on this method name:
    List<Customer> findByName(String name);
}

Step 4: Exposing REST Endpoints

Expose REST endpoints to perform write and read operations against the database.

Create the Controller Class

Create a class named CustomerController.java and wire in the repository:

CustomerController.java
Copied! 
package com.example.spanner.controller;

import com.example.spanner.model.Customer;
import com.example.spanner.repository.CustomerRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
@RequestMapping("/customers")
public class CustomerController {

    @Autowired
    private CustomerRepository customerRepository;

    @PostMapping
    public String saveCustomer(@RequestBody Customer customer) {
        // Saves the entity to Spanner. 
        // Under the hood, this borrows a write-prepared session from the pool.
        customerRepository.save(customer);
        return "Customer successfully saved!";
    }

    @GetMapping("/{id}")
    public Customer getCustomer(@PathVariable Long id) {
        return customerRepository.findById(id).orElse(null);
    }

    @GetMapping("/search")
    public List<Customer> getCustomersByName(@RequestParam String name) {
        return customerRepository.findByName(name);
    }
}
