Step 1: Architecture & Solution Overview

Learn how Java applications connect to Google Cloud Spanner using Spring Data JPA, Hibernate PostgreSQL Dialect, and PGAdapter.

What is PGAdapter?
Google Cloud Spanner PGAdapter is a lightweight sidecar proxy that converts PostgreSQL wire-protocol (v3.0) SQL statements from standard PostgreSQL JDBC drivers into Spanner gRPC API calls in real time.
Why Spring Data JPA + PGAdapter?
Allows Enterprise Java applications to leverage standard JPA/Hibernate annotations (`@Entity`, `@Transactional`, `@Repository`, `saveAll`) while storing data in horizontally scalable, multi-region Google Cloud Spanner.

Step 6: JPA Entities & Anti-Hotspot Primary Keys

Design domain entities using UUID primary keys to prevent write hotspots on Spanner splits.

CRITICAL RULE: Avoid Auto-Increment / Sequence IDs
NEVER use GenerationType.IDENTITY or AUTO_INCREMENT sequences in Spanner! Sequential numbers cause all new writes to land on a single split node (hotspotting). Always generate random UUIDs or bit-reversed keys.
Customer Entity (src/main/java/.../model/Customer.java)
Copied! 
@Entity
@Table(name = "customers")
public class Customer {

    @Id
    @Column(name = "customer_id", length = 36, nullable = false)
    private String customerId;

    @Column(name = "full_name", nullable = false)
    private String fullName;

    @Column(name = "email", nullable = false)
    private String email;

    @PrePersist
    public void ensureUUID() {
        if (this.customerId == null) {
            // Anti-Hotspot: Use random UUIDs to distribute writes across all Spanner splits
            this.customerId = UUID.randomUUID().toString();
        }
    }
}

Step 8: Snapshot Reads & Read-Only Transactions

Use @Transactional(readOnly = true) for read methods to execute non-locking Snapshot Reads in Spanner.

Spanner Performance Tip: Snapshot Reads
In Cloud Spanner, Read-Only transactions do NOT acquire locks and do NOT block concurrent writes. Marking Spring services with @Transactional(readOnly = true) tells PGAdapter to execute snapshot reads, boosting system QPS!
BankingService.java (Read-Only Methods)
Copied! 
@Service
@RequiredArgsConstructor
public class BankingService {

    // BEST PRACTICE: Snapshot Read - Non-locking, highly scalable
    @Transactional(readOnly = true)
    public List<Customer> getAllCustomers() {
        return customerRepository.findAll();
    }
}

Step 9: Atomic Read-Write Transactions & Bulk Batch Writes

Implement serializable money transfers and high-speed batch operations with saveAll().

BankingService.java (Write & Batch Methods)
Copied! 
// 1. Atomic Money Transfer Transaction
@Transactional
public PaymentTransaction transferMoney(String sourceId, String targetId, BigDecimal amount) {
    Account source = accountRepository.findById(sourceId).orElseThrow();
    Account target = accountRepository.findById(targetId).orElseThrow();

    source.setBalance(source.getBalance().subtract(amount));
    target.setBalance(target.getBalance().add(amount));

    accountRepository.save(source);
    accountRepository.save(target);

    PaymentTransaction txn = PaymentTransaction.builder()
            .sourceAccountId(sourceId).targetAccountId(targetId)
            .amount(amount).status("COMPLETED").build();
    return paymentTransactionRepository.save(txn);
}

// 2. High-Performance Bulk Batch Insertion (Customers + Accounts)
@Transactional
public List<Customer> seedBulkCustomers(int count) {
    List<Customer> customerList = new ArrayList<>();
    List<Account> accountList = new ArrayList<>();

    for (int i = 1; i <= count; i++) {
        String customerId = UUID.randomUUID().toString();
        Customer customer = Customer.builder().customerId(customerId)
                .fullName("User " + i).email("u" + i + "@bank.com").status("ACTIVE").build();
        customerList.add(customer);

        Account account = Account.builder().accountId(UUID.randomUUID().toString())
                .customerId(customerId).accountType("CHECKING").currency("USD")
                .balance(BigDecimal.valueOf(1000.00)).build();
        accountList.add(account);
    }
    // saveAll triggers JDBC batching (hibernate.jdbc.batch_size=50) for both tables
    List<Customer> saved = customerRepository.saveAll(customerList);
    accountRepository.saveAll(accountList);
    return saved;
}

Step 11: Application Engineer Best Practices Summary

Essential guidelines for building enterprise Java applications on Cloud Spanner.

1. Anti-Hotspotting PKs
Never use sequence or auto-increment IDs! Use random UUIDs or hash-prefixed keys so writes distribute across Spanner splits.
2. Snapshot Reads
Annotate all read methods with `@Transactional(readOnly = true)` to avoid acquiring locks and eliminate read/write lock contention.
3. JDBC Bulk Batching
Configure `hibernate.jdbc.batch_size: 50` and `order_inserts: true` to bundle multiple DML statements into single network RPCs.
4. Connection Pool Tuning
Set HikariCP pool size (30–50) with `connection-test-query: SELECT 1` and `max-lifetime: 1800000` to manage PGAdapter proxy connections.
