Lab 0: Installing and Configuring Docker & Google Cloud CLI

Welcome to the prerequisite setup lab. Before building applications with Google Cloud Spanner, you need to prepare your machine by installing Docker Desktop and the Google Cloud CLI (gcloud SDK).

Why Do We Need These Tools?

Developing with Cloud Spanner locally is optimized by using two main infrastructure components:

Docker Desktop
Hosts the Spanner Emulator container locally. This runs a simulated in-memory version of Cloud Spanner on your computer without incurring cloud costs.
Google Cloud CLI
Command-line interface used to manage Cloud Spanner instances, databases, and schemas, and run DDL/DML statements.
Supported Operating Systems
This setup guide covers step-by-step installations for both macOS (Intel & Apple Silicon) and Windows 10/11. Select the corresponding platform tab in each step.
  • Read the Setup Objectives Review why we need Docker Desktop and gcloud CLI for development.

Step 1: Installing Docker Desktop

Docker is required to run the Cloud Spanner Emulator container. Choose your Operating System from the tabs below to proceed with the installation.

System Requirements

  • Windows 10/11 64-bit: Home, Pro, Enterprise, or Education.
  • Enable **WSL 2 (Windows Subsystem for Linux)** or Hyper-V features.
  • Virtualization enabled in BIOS settings.

Installation Options

Option 1: Graphical Installer (Recommended)

  1. Download the installer from the official Docker Desktop Windows Installer.
  2. Double-click Docker Desktop Installer.exe to run it.
  3. Ensure the **"Use WSL 2 instead of Hyper-V"** checkbox is selected during setup.
  4. Follow the installer wizard and restart your computer when prompted.

Option 2: Windows Package Manager (Winget)

Open PowerShell as Administrator and execute:

PowerShell (Admin)
Copied! 
winget install Docker.DockerDesktop

Determine Your Mac Chipset

Open the Apple Menu  in the top-left corner and click "About This Mac". Note whether you have an **Intel** processor or **Apple Silicon** (M1/M2/M3/M4 chips).

Installation Options

Option 1: Official Installer (DMG)

  • Download DMG for Apple Silicon Mac (M-series chips).
  • Download DMG for Intel Mac.
  • Open the Docker.dmg file and drag the **Docker** icon into your **Applications** folder.

Option 2: Homebrew

If you use Homebrew package manager, run the following command in terminal:

macOS Terminal
Copied! 
brew install --cask docker
Start the Application
After installation, you **must open** Docker Desktop to start the background daemon. On Windows, search for "Docker Desktop" in the Start Menu. On Mac, open it from Applications or Spotlight. You will see a small whale icon in your taskbar/status bar once it is ready.
  • Install Docker Desktop Download and run the installer for Windows or macOS.
  • Start Docker and complete initialization wizard Verify that the Docker service shows green ("Running") status.

Step 2: Installing Google Cloud CLI (gcloud)

The Google Cloud CLI contains the tools and SDK engines required to interact with Google Cloud resources and manage your local database emulator.

Installation Procedure

  1. Download the official gcloud CLI Windows Installer.
  2. Launch `GoogleCloudSDKInstaller.exe` and follow the configuration wizard.
  3. During the installation options page, ensure the **"Single User"** or **"All Users"** setup completes.
  4. Keep **"Start Google Cloud SDK Shell"** and **"Run gcloud init"** checked at completion.
CLI Path Resolution
The Windows installer automatically appends gcloud paths to your environment variables. If you get a "gcloud: command not found" error, restart your terminal.

Option 1: Graphical/Interactive script (Recommended)

Open a terminal window and run the following curl sequence to fetch and install the SDK:

macOS Terminal
Copied! 
curl https://sdk.cloud.google.com | bash

This script prompts you to add path environment configurations to your default shell configuration profile (.zshrc or .bash_profile). Respond **Yes** to all prompts.

Option 2: Homebrew

If you manage packages using Homebrew:

macOS Terminal
Copied! 
brew install --cask google-cloud-sdk
  • Install Google Cloud SDK Install the gcloud package and reload your terminal session.

Step 3: Initializing and Configuring gcloud

Before running commands against actual cloud products or development emulators, you must initialize the gcloud CLI with your identity and select a default project.

1. Run gcloud Initialization

Open a terminal window (on Windows, use Command Prompt or PowerShell; on macOS, use Terminal) and run:

Terminal (Bash / PowerShell)
Copied! 
gcloud init

2. Complete Authorization Walkthrough

The initialization wizard will walk you through these options:

  1. Authentication Login: It will ask to log you in. Type Y and press enter. A web browser window will open. Log in with your Google account.
  2. Select Cloud Project: Choose an existing Google Cloud project from the menu list or type in a project ID to use as default. (If you don't have one, select the option to create a project or type in a placeholder like my-spanner-project).
  3. Default Compute Region: If asked to choose default zones/regions, select one close to you (e.g. us-central1 or europe-west3).
Already Logged In?
If you have configured gcloud in the past, you don't have to reinitialize. You can verify your active account profile by running gcloud config list.
  • Run gcloud init and log in Complete the authentication login process in your browser.
  • Establish default project configs Confirm that your active configuration has a default project assigned.

Step 4: Verifying the Setup

Now that both tools are installed, let's verify that they are correctly added to your system's PATH variables and can be executed from the command line.

1. Verify Docker Engine

Open a terminal and type the following command to check if the Docker command-line tool is reachable and can connect to the background engine:

Terminal (Bash / PowerShell)
Copied! 
docker --version
docker info

If successful, docker info will print a summary of your running container counts, kernel versions, and storage configs. If you receive an error about "cannot connect to the Docker daemon", make sure **Docker Desktop is open and running**.

2. Verify gcloud SDK

Run the following command to print details about your installed Cloud CLI version and its components:

Terminal (Bash / PowerShell)
Copied! 
gcloud --version

Verify that the output displays the main `Google Cloud SDK` package along with secondary utilities (e.g. `bq`, `gsutil`, `core`).

  • Run Docker validation commands Confirm that docker is reachable and the engine is in a running state.
  • Run gcloud validation commands Confirm that gcloud outputs version numbers successfully.
