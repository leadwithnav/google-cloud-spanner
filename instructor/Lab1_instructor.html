Instructor Lab Guide

Lab 1: Banking System Setup in Spanner, Emulator Basics, and DDL/DML

This guide is meant for the instructor to run the lab smoothly, anticipate learner issues, and keep the session on pace using a banking scenario.

Lab Overview

Goal: help learners create a Google Cloud Spanner instance with 100 PU, start the emulator, and run basic DDL and DML operations for a simple banking application.

Audience: beginners who are new to Spanner and want a practical first experience.

Duration: 30–45 minutes.

  • Provision a Spanner instance with 100 PU or use the emulator.
  • Create a database and banking-focused schema.
  • Insert, read, update, and delete sample account and transaction data.
  • Explain the differences between schema setup and data operations.

Instructor Preparation

  • Ensure learners have a Google Cloud account or emulator access.
  • Confirm the Google Cloud CLI is installed and authenticated.
  • Have the emulator commands ready before the session.
  • If learners hit auth issues, pause and troubleshoot before moving on.

Suggested Demo Commands

gcloud config set project YOUR_PROJECT_ID

gcloud spanner instances create banking-instance \
  --config=regional-us-central1 \
  --description="Banking lab instance" \
  --processing-units=100

gcloud spanner databases create banking-db \
  --instance=banking-instance \
  --ddl='CREATE TABLE Customers (CustomerId STRING(36) NOT NULL, FullName STRING(100) NOT NULL, Email STRING(255), CreatedAt TIMESTAMP NOT NULL, PRIMARY KEY (CustomerId))'
Instructor tip: if you are using the emulator, keep the setup commands shorter and focus on the banking data lifecycle rather than cloud-specific details.

Run Spanner Emulator with Docker

# Start the emulator container
docker run --rm -d --name spanner-emulator \
  -p 9010:9010 -p 9020:9020 \
  gcr.io/cloud-spanner-emulator/emulator:latest

# Create a new local gcloud configuration profile
gcloud config configurations create emulator

# Set the endpoint override to route Spanner commands to the local emulator
gcloud config set api_endpoint_overrides/spanner http://localhost:9020/

# Set a dummy project ID (required by gcloud commands but ignored by the emulator)
gcloud config set project emulator-project

# Disable credentials since the local emulator doesn't require authentication
gcloud config set auth/disable_credentials true
Instructor tip: after the emulator is running, learners can create a database and run the banking DDL/DML commands exactly as they would against a real instance.

Banking Schema Example

CREATE TABLE Customers (
  CustomerId STRING(36) NOT NULL,
  FullName STRING(100) NOT NULL,
  Email STRING(255),
  CreatedAt TIMESTAMP NOT NULL,
) PRIMARY KEY (CustomerId);

CREATE TABLE Accounts (
  AccountId STRING(36) NOT NULL,
  CustomerId STRING(36) NOT NULL,
  AccountType STRING(20) NOT NULL,
  Balance NUMERIC,
  Status STRING(20) NOT NULL,
) PRIMARY KEY (AccountId);

INSERT INTO Customers (CustomerId, FullName, Email, CreatedAt)
VALUES ('C1001', 'Aisha Khan', 'aisha.khan@example.com', CURRENT_TIMESTAMP());

INSERT INTO Accounts (AccountId, CustomerId, AccountType, Balance, Status)
VALUES ('A1001', 'C1001', 'Savings', 15000, 'Active');

SELECT * FROM Customers WHERE CustomerId = 'C1001';

UPDATE Accounts
SET Balance = 16000
WHERE AccountId = 'A1001';

Common Issues and Fixes

  • Auth issue: run gcloud auth login and verify the active account.
  • Project not found: confirm that the project exists and is selected with gcloud config set project.
  • Instance creation failed: check the region, quota, or API enablement.
  • Emulator not reachable: verify that the emulator process is running and that the environment variable points to the correct endpoint.

Suggested Discussion Questions

  • Why do we need a primary key in Spanner for a banking table?
  • How is DDL different from DML in the context of accounts and customers?
  • What happens when a customer record exists but no account record is linked to it?
  • Why might you prefer the emulator for learning before using a real instance with 100 PU?

Wrap-Up Notes

Conclude the lab by reminding learners that this first exercise is about getting comfortable with the workflow: create, define schema, insert data, and query it. That foundation will make later labs on optimization and design much easier.
