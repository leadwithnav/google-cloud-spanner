Step 8 Role Inheritance ~10 min
17 Create role hierarchy: employee_reader_role → manager_role DDL ✓ Should Succeed
employee_reader_role gets SELECT on 4 cols. manager_role gets SELECT on Salary directly, then inherits employee_reader_role via GRANT ROLE. manager_role ends up seeing 5 cols total. 
gcloud spanner databases ddl update test-database1 \
  --instance=test-instance1 \
  --ddl="CREATE ROLE employee_reader_role;
   CREATE ROLE manager_role;
   GRANT SELECT (EmployeeId, Name, Email, Department)
     ON TABLE Employees TO ROLE employee_reader_role;
   GRANT SELECT (Salary)
     ON TABLE Employees TO ROLE manager_role;
   GRANT ROLE employee_reader_role TO ROLE manager_role;"
✓ Expected Output
Schema was updated successfully.

Step 9 Row Filtering Through Views ~10 min
20 Create engineering_role, definer-rights view, GRANT on view DDL ✓ Should Succeed
SQL SECURITY DEFINER means the view executes with the schema owner's privileges, not the caller's — so engineering_role sees Engineering rows through the view without needing base-table access. 
gcloud spanner databases ddl update test-database1 \
  --instance=test-instance1 \
  --ddl="CREATE ROLE engineering_role;
   CREATE VIEW EngineeringEmployees
     SQL SECURITY DEFINER AS
     SELECT EmployeeId, Name, Email, Department
     FROM Employees
     WHERE Department = 'Engineering';
   GRANT SELECT ON VIEW EngineeringEmployees
     TO ROLE engineering_role;"
✓ Expected Output
Schema was updated successfully.
21 SELECT * FROM EngineeringEmployees as engineering_role QUERY ✓ Should Succeed
Should return only 2 rows (Bob, Alice) — the Engineering department filter hides Charlie (Sales).
gcloud spanner databases execute-sql test-database1 \

  --database-role=engineering_role \
  --sql="SELECT * FROM EngineeringEmployees"
✓ Expected Output — 2 rows (Engineering only)
EmployeeId Name Email Department 201 Bob Miller bob.miller@company.com Engineering 202 Alice Croft alice@company.com Engineering
